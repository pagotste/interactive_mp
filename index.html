Self evaluation

Lab 1

Lab 1 was quite easy because I have already done some web design with basic CSS and HTML. I learned a bit more about JavaScript though, because I haven’t done much JS in the past. I also was quite familiar with the definitions so it didn’t take me lot of time to do them.

Lab 2

For this lab, I was familiar already with responsive web design so I was able to make the pages quite fast. I didn’t know about em though, so I learned about that. Overall, I didn’t have any difficulties with this lab.

Lab 3

For the chat, I didn’t know at all about web sockets so that was new knowledge for me. Thanks to the live coding, I was able to do it quite fast. The MVC person list was a bit more difficult because I had issues with the web storage. Now I kinda know how it works. But the MVC part wasn’t the hardest thing to do because I already worked with a MVC framework (CodeIgniter) even though it was PHP. The web radio was okay, not too hard and not too easy, but I didn’t know about how it worked so I learned more about it.

Lab 4

The small game was easier than the draw app. I had issues with the guidance line with the draw app because it would either not be a line or not deleting the previous guidance line. I worked with a friend from this course and exchanged information to finally manage to have a working result. The small game was mostly thread handling, something I already did in Java, so adapting this knowledge to JavaScript made the task easier.

Lab 5

I never used jQuery and AJAX, so I learned about that. Adapting the person list wasn’t too difficult. The simple app using AJAX also went quite easily. Overall this lab wasn’t too hard.

Lab 6

I never used flickr, but I have used an API (Google Maps) for another course so I was familiar about API usage. However, I had a hard time making the grid work correctly with the bootstrap I used, even though I already used ne in the past. So what took me the most time was design rather than JavaScript.

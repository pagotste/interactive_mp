Responsive Web design


Answer to questions

What is the purpose of the RWD? Why it is used? How things are done before RWD? What is the difference?

The purpose of responsive web design is to make web pages adapt to the device used to display a web page, without the need of a different web page/HTML code specific to the device, and with the same content on every device thanks to its size being defined with relative sizing and/or media queries defined according on the screen resolution.

It is used because of the various screen resolutions that a user can have. For example, before RWD was used, everything had its size defined with pixels or inches. But on a screen with a width of 380px, an element wider than this size can’t be displayed properly. Some text elements can also become cropped and so on. Now if the element’s width was defined in percentages, say 80%, it would have a width of 304px.

How css media queries can help you building responsive web sites?

CSS media queries can be used to detect the screen resolution of the device the web page is displayed on. Thanks to that, we can define the CSS we want to use for this resolution, to display every element or change the way to display some elements (a menu for example).

Why to use em (font) based sizing for example margins?

Em based sizing is relative to the font size the element inherited. That means that instead of having a set size, the element can have its size change depending on the base size. If we take margins as an example, if we define it in px the margin will keep the same size on every device, so it could affect the way text is displayed and move some elements around. If the margin is defined in em, the margin will be relative to the font size, in other words, relative to the screen size. That would mean the elements stay where they are supposed to be.

Article is already 7 years old, do we have any new technology which can be used to achieve same type of UI responsiveness as described in the article.

Nowadays, we can use CSS Grid (2 dimensions)/Flexbox (1D) to add in responsiveness by using fluid containers to put content in. These elements adapt to the screen size of the device and change the inner elements’ layout according to it. Grids are used mostly for wide layouts while Flexbox is intended for simpler layouts.

How could you prevent user zooming the site on mobile?

To prevent the user from zooming, we can use a meta tag with the properties width=device-width, initial-scale=1, maximum-scale=1, user-scalable=0 (see meta tags of this page).

HTML5 specification introduces new semantic elements. One of them is a figure element. Can you say why you should use it in addition to standard img elements?

A figure markup should be used because it can contain more than one element, such as a caption for the figure contained, or a list of images. Also, a figure markup is not restricted to images, it can contain several types of media elements (video, audio, code…).
